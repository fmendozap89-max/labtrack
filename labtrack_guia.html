Abrir LabTrack en el navegador

2

Asignar múltiples analistas a un mismo lote

Podés agregar más de un analista. Cada analista ve su propia parte asignada. El estado del producto pasa a Parcial cuando algunos completaron pero otros no.

ℹ Si asignás un analista adicional a un lote ya Listo p/revisar o Revisado, el estado vuelve automáticamente a En análisis — el sistema requiere que la nueva parte también se complete.
3

Filtrar y buscar

Usá los botones de filtro rápido (Pendientes · En análisis · Vencidos · Completados) o el buscador por producto/lote. Las tarjetas de estadísticas del panel también actúan como filtros.

4

Editar o replanificar — ícono ✎ en cada fila

Abre el modal de edición. Podés cambiar analistas, fechas, horario estimado y prioridad. La opción Replanificar genera una nueva asignación si el analista no pudo completar la tarea original.

9 · Sincronización GitHub Gist (multi-usuario)
1

Generar token en GitHub

Ir a github.com → Settings → Developer settings → Personal access tokens. Crear token con permiso gist. Copiar el token (solo se muestra una vez).

2

Configurar en LabTrack — botón "⚙ Configurar sync" en la barra superior

Ingresar el token y un nombre para identificar el Gist. LabTrack crea o localiza el Gist automáticamente.

⚠ Al conectar Gist por primera vez, los datos del sistema se tomarán del Gist (no del almacenamiento local). Asegurate de que el Gist tenga el estado correcto antes de conectar.
3

Funcionamiento en modo sincronizado

Cada operación de guardado sube los cambios al Gist. Cuando dos usuarios trabajan simultáneamente, el sistema aplica un merge por entidad (por análisis, por solicitud, etc.) para evitar pérdida de datos. El badge en la barra superior indica el estado: ✓ online · ⚠ offline · ↻ sincronizando.

Día 2 de atraso — aviso al analista

Si la fecha programada de una tarea pasó hace 2 días y no hay horas registradas, el sistema muestra un banner naranja al inicio de "Mi día" y un borde naranja en la fila de la tarea. La banda de aviso indica exactamente qué hacer.

⚠ Tenés 1 día de margen antes de que el supervisor reciba una alerta automática. Completá el análisis o solicitá replanificación.
!

Día 3 de atraso — supervisor notificado

A los 3 días de atraso el supervisor ve la alerta en su 🔔 de notificaciones. Puede completar la tarea por vos (ingresando las horas reales), replanificar la fecha o eliminar la tarea. Actuá antes si es posible para registrar el trabajo vos mismo.

2 · Registrar trabajo — Mis análisis
1

Abrir la tarea asignada

Hacé clic en la fila de la tarea en Mis análisis. Se abre el panel de detalle con el historial y los controles de registro.

2

Registrar horas o sesiones

Si el análisis se realiza en un solo bloque: ingresá las horas reales directamente.
Si se divide en varios días: usá Agregar sesión por cada jornada, indicando fecha y horas de esa sesión. El total se calcula automáticamente.

✓ Podés agregar notas por sesión para registrar observaciones del proceso (temperatura, incidencias, etc.).
3

Marcar como completado

Cuando terminaste, cambiá el estado a "Listo para revisar". El revisor y supervisor reciben la notificación. El producto pasa a estado Listo p/revisar.

⚠ Si el lote tiene múltiples analistas, el estado queda en Parcial hasta que todos completen su parte.

❓ Soporte

Problemas frecuentes

Soluciones a las situaciones más comunes. Clic en cada problema para expandir.

🔄Recargué el navegador y no veo mis datos▼
LabTrack guarda una copia local automática. Si los datos desaparecen:
  • Modo incógnito: el almacenamiento local no persiste — usá una ventana normal.
  • Caché borrado: si limpiaste datos del navegador, la copia local se eliminó.
  • Gist configurado: al recargar con conexión a internet, los datos se restauran del servidor automáticamente. Verificá la conexión y el estado de sincronización (badge en la barra superior).
  • Si el badge de sync muestra error, el token de GitHub puede haber expirado. El supervisor debe generar un nuevo token.
🔑Contraseña incorrecta / no puedo ingresar▼
  • Verificá que Bloq Mayús no esté activado.
  • La contraseña es sensible a mayúsculas y minúsculas.
  • Si olvidaste tu contraseña, el supervisor puede restablecerla desde Administración → Usuarios → columna Nueva contraseña.
  • Si sos supervisor y perdiste acceso, la contraseña por defecto de la instalación es la que se configuró inicialmente.
🧪No veo las tareas que me asignaron▼
  • Asegurate de estar en Mis análisis, no en el Panel general.
  • Verificá con el supervisor que la tarea está asignada a tu usuario específico.
  • Si el lote tiene estado Parcial, puede que solo parte esté asignada a vos — buscá por nombre de producto.
  • Recargá la página para sincronizar los últimos cambios del servidor.
📅No puedo solicitar cambio de horario▼
  • Solo podés tener una solicitud de cambio de horario pendiente por día. Si ya enviaste una, el botón muestra ⏳ Cambio pend. y no se puede enviar otra hasta que el supervisor la resuelva.
  • Sin embargo, podés enviar simultáneamente una solicitud de horas extra para el mismo día — son tipos independientes.
  • Si el supervisor rechazó tu solicitud anterior, podés enviar una nueva.
🔴Mi análisis volvió a estado "Parcial" o "En análisis" sin que yo hiciera nada▼
Esto es comportamiento esperado, ocurre en dos situaciones:
  • El supervisor asignó otro analista al lote: el estado vuelve a Parcial porque hay nueva trabajo pendiente por completar.
  • El revisor observó el análisis y solicitaste replanificación: el estado vuelve a Parcial/En análisis hasta que completés la nueva tarea generada.
Revisá Mis análisis para ver si hay nuevas asignaciones activas.
🔄Varios usuarios trabajamos al mismo tiempo — ¿se pueden perder datos?▼

Con Gist configurado, el sistema usa una estrategia de merge inteligente: cuando dos usuarios guardan cambios casi simultáneamente, se fusionan por entidad (cada análisis, cada solicitud, etc.) preservando ambos cambios. El último guardado prevalece solo si modifica la misma entidad.

Sin Gist: los datos solo existen en el navegador local de cada usuario y no se comparten. No es apto para trabajo multi-usuario simultáneo.

⚙️El badge de sync muestra error de conexión▼
  • Verificá la conexión a internet.
  • El token de GitHub puede haber expirado. El supervisor debe generar uno nuevo en GitHub → Settings → Developer settings → Personal access tokens y actualizarlo en Configurar sync.
  • En modo offline, los datos se guardan localmente y se suben automáticamente cuando vuelve la conexión.
  • Usá Diagnóstico (en Administración → Sincronización) para ver el detalle del error.
🕐La sesión se cerró sola▼
Por seguridad, la sesión cierra automáticamente tras 30 minutos de inactividad o al cumplirse 12 horas desde el inicio de sesión. Ingresá nuevamente — todos los datos están intactos.
🗑️Necesito resetear todos los datos del sistema▼
Solo el supervisor puede hacer esto desde Administración → Reinicio de datos operativos. Esta acción elimina análisis, solicitudes y actividades de gestión pero conserva usuarios, contraseñas y horarios laborales.

⚠ Es irreversible si no tenés un backup. Antes de reiniciar, exportá los datos desde Administración → Exportar JSON.
✅Si el problema no está en esta lista o persiste, toda la actividad queda registrada en el Audit Trail (visible para el supervisor). Compartí el log con el administrador del sistema para diagnóstico.
